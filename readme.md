Mise en pratique
Par équipe de 4 :

Dev1 :
Créé un nouveau projet en local, l'initialise et créé une base de fichiers (readme.md, index.html, global.css).
Créé un repository à distance sur Github, établi le lien remote, push tout sur la main.
Invite les autres dev en collaborateurs
Créer une branche dev, et rajoute une TODO-List dans le readme de qui doit faire quoi. (une page d'accueil, une page à propos, une page contact, une fausse page connexion), et la push.

Le Dev1 s'attaque ensuite à sa feature home page

Dev2 :
Clone le projet et, depuis la dev, créé sa branche pour faire sa feature about page

Dev3
Clone le projet et, depuis la dev, créé sa branche pour faire sa feature contact page

Dev4
Clone le projet et, depuis la dev, créé sa branche pour faire sa feaure page login

Important :
Vous devrez tous·tes modifier le MÊME fichier css pour faire un max de conflits 😈 (Vous pouvez vous concerter avant pour les choix design à respecter)
Quand vous avez fini votre feature, modifiez le readme pour indiquer dans la todolist que la tâche est faite
Développez un minimum les pages qu'il y ai de la matière sans trop en faire.
Les pages ne doivent pas être fonctionnelles (osef du formulaire de contact qui envoie rien à la fin, la page login renvoie juste vers la page d'accueil)
Thème de votre choix pour le site, tant que ça reste SFW.

Finalité
Chacun merge manuellement son travail à la branche dev et vous gérez les conflits ensemble.
Ensuite, ensemble, faites une pull request pour ramener la dev sur main.
